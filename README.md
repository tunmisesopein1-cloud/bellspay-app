This report explains the frontend development of student digital
bank. The rapid growth of digital technology has transformed the wat
financial services are delivered. In recent years, the banking sector has
experienced significant changes due to advancement in information
and communication technology. Digital allows users to perform
banking activities such as transferring funds, checking account
balances, and making payments through online platforms without
visiting a physical bank. This project presents the design and
implementation of the frontend of a student digital bank, focusing on
simplicity, responsiveness and an improved user experience.
This project’s objective is to design a simple user interface that
allows students navigate the site easily. By developing a fully
responsive design, BellsPay app works properly on mobile phones to
ensure a smooth user experience. It’s build mainly on speed and easy
access and a well organized interface.
BellsPay is a web-based digital payment application built to ensure basic every
day financial transactions simpler and more convenient for users. Key features
like login page and dashboard are easy to find, helps users go through the
platform without confusion.
Through thoughtful reasoned design and solid fronted development practices,
BellsPay shows how a digital platform can be built to clearly and effectively
meet users needs.
Tools Used
• HTML (Hypertext Markup Language): Used to structure the content and
layout of the web pages. It was used in arranging elements like forms, buttons,
navigation bars and text content.
CSS (CASCADING STYLE SHEET): Used for styling, layout design, and
improving the visual appearance of the application. It was used to style theeeb
application and improve its overall appearance. It was im charge of the color, fonts,
spacing and layout positioning. It makes the website responsive so that it displays
well on mobile devices and PCs.
• REACT: Used to build reusable components and manage the user interface
efficiently. It made it simpler to build a reusable components and manage how
different parts of the website works.
• FIGMA: Used for designing the user interface and planning the overall layout before
development. It was used during the design phase to plan the user interface before the
development starts.
• SQL (STRUCTURED QUERY LANGUAGE) : Used for managing and storing
user data in the database. It enables data to be added and updated and collected
efficiently, ensuring proper management and consistency in the system.
PROBLEMS ENCOUNTERED
During the development of this site, several obstacles were encountered;
• WEBSITE CRASHING: The website experienced lots of crashes due to
errors in the code and improper Components.
• CSS STYLING ERRORS: Some elements didn’t display correctly cause
of conflicting CSS rules.
• RESPONSIVENESS ISSUES: The website initially did not adapt
properly to different screen sizes, especially on mobile devices.
• NAVIGATION ISSUES: The navigation between pages was not smooth.
Some links didn’t respond as expected, and pages transistion were slow
due to routing and components issues
• DEBUGGING ISSUES: Identifying the source of errors was time
consuming, especially when multiple components were affected.
SOLUTIONS IMPLEMENTED
To address the problems encountered, the following solutions were applied;
• DEBUGGING THE CODE: The code base was carefully examined to
identify and fix bugs responsible for crashes while running the code
• CSS ERROR FIXING: Styling issues where corrected by reviewing and
adjusting CSS rules
• USE OF AI TOOLS: Github Copilot and AI tools were used to help fix
CSS styling errors and improving layout responsiveness.
KEY FEATURE OF THE WEBSITE
The BellsPay Web includes the following features;
• SMOOTH NAVIGATION: Users can easily move between different
sections of the websites without difficulty.
• CLEAR ICONS:Icons are well designed and easy to understand,
improving user experience.
• ACCESSIBILITY AND SPEED: The website is fast and accessible,
ensuring efficient user interaction.
• EASY LOGIN ACCCESS: The login section is easy to locate and use,
improving usability for first time users.
• RESPONSIVE DESIGN: The website is fully responsive and adapts
properly to different screen sizes, including mobile phones, tablets and
desktop systems.
The Bellspay Web focused on the designing of a responsive, user friendly e-
banking. Some challenges were faced during the development of the website but
with careful debugging all errors were resolved.
